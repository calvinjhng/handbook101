Lab 4

note

The following document assumes a Linux environment. If this is unavailable to you, you may consider running an EC2 instance.

Please exercise care when copying code. Explanations are provided as much as possible but feel free to raise any clarifications.

1a. Install Python (if required)#

# Check for python3
python3 --version

Check if you have Python installed.

# Install Python3
sudo apt install python3

If Python is not installed, run the above command to install it.

1b. (Optional) Use virtual environments#

# Usage:
python3 -m venv <NAME_OF_VIRTUAL_ENVIRONMENT>
# Example: In this case, the virtual environment is simply named 'env'
python3 -m venv env

Virtual environments help you to manage different applications which you may be developing at the same time that might utilise different versions of the same libraries. Because you might require different versions of the same libraries for different applications, virtual environments allow you to avoid breaks introduced by unintentionally upgrading/downgrading one or more libraries as you move from developing one application to another.

If successful, your command prompt will be prefixed with env or whatever you have chosen to name your virtual environment.

sudo apt-get install python3-venv

In case venv is not installed, run the above command to install it.

# Usage:
source <NAME_OF_VIRTUAL_ENVIRONMENT>/bin/activate
# Example:
source env/bin/activate

To enter a virtual environment where you get a segregated set of libraries or dependencies, run the source command.

To exit a virtual environment, simply run the deactivate command.

1c. Install Flask (if required)#

# Check for python3
flask --version

Similarly, check if Flask is installed.

# Install flask
sudo apt install python3-flask

If not, follow the prompt and install it accordingly.

2. Write your first Flask application#

from flask import Flask
app = Flask(__name__)
@app.route("/")
def index():
return "Welcome to Yumyum!"
if __name__=='__main__':
app.run(debug=True, host='0.0.0.0')

Create a new file application.py and write the above contents into it.

3a. Set environment variables for the Flask application#

export FLASK_APP=application.py
export FLASK_ENV=development

Set the variables FLASK_APP and FLASK_ENV as above.

3b. Run your Flask application#

flask run

Upon running the above, you should get a message noting that your Flask app is being served.

Further, by navigating to the link indicated in the message (the default should show something like 127.0.0.1:5000), one will be greeted by the same message as set in the application.py file.

Returning to the command prompt, it is safe to terminate the running process with Ctrl C as we will be going to create a Docker container image next.

4a. Create requirements.txt#

Create a file called requirements.txt and include strictly only one line saying Flask==1.1.2

4b. Create your Dockerfile#

FROM python:3.7
RUN apt-get update -y && \
apt-get install -y python
COPY ./requirements.txt /app/requirements.txt
WORKDIR /app
RUN pip install --upgrade pip
RUN pip install -r requirements.txt
COPY . /app
ENTRYPOINT ["python"]
CMD ["application.py"]

Create another file called Dockerfile with the above content.

5a. Build your Docker Image.#

With the prior steps completed, run docker build -t yumyum-flask:latest .

Due to recent changes in security posture, you might find it neccesary to include sudo in front of your command.

Note that this command:

  • Builds your Docker Image
  • Names it yumyum-flask
  • Tags it as the latest version

5b. Run your new image#

docker run -d -p 5000:5000 yumyum-flask

  • The run command instructs the Docker runtime environment to run a given image. In this case, it is the yumyum-flask image that you have just built.
  • The -d option indicates that the container should be run in detached mode. Put simply, it runs in the background and does not hold up your terminal or command prompt.
  • Using the -p command, we are able to specify the host system port mapping (left) to the application port (right). The application port is usually specified by the developer and the left port number is what you would usually want to pay attention to as this is the port through which you will access the application.

6. Verify your new image is indeed working#

After running the command in the prior step, you should be able to see the same website at the same address (by default, this is 127.0.0.1:5000).

Once you have confirmed this, it is safe to stop your running container.

  • Run docker container ls to list out all running containers. The container you launched should be in the list.
  • Run 'docker container rm -f <ID_OF_CONTAINER>`
    • The -f option forcefully removes a container, even if it is running.
    • The container ID does not need to be spelt out in full. A minimum length of 2 is sufficient as long as it can be differentiated from any other containers.
